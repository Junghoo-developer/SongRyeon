# Administrative Reform 1

**별칭**: 행정 개혁 1호  
**목적**: SongRyeon Core의 문서, 설계, 발주, 실행 기록을 헷갈리지 않게 나누기 위한 날먹 행정 체계.

## 계층

| 번호 | 폴더 | 계층 | 의미 |
|---:|---|---|---|
| 0 | `00_Philosophy/` | 철학 | 애매한 생각, 미승격 개념, 아직 실행 권한이 없는 문제의식 |
| 1 | `01_Maintenance_System/` | 영구 유지 체계 | 에이전트 협업 체계, 프로젝트 운영 방침, 문서 승격 규칙 |
| 2 | `02_Constitutions/` | 헌법 | 큰 변화가 있을 때 이전 모습과 앞으로의 모습을 대조하며 비전 제시 |
| 3 | `03_Maps/` | 지도 | 헌법 지도, 기능 지도, 개발 지도 |
| 4 | `04_Orders/` | 발주서 | 개발 지도에서 내려온 구체적 구현 지시 |
| 5 | `05_Execution_Records/` | 실행 기록 | 대화 기록, 턴 회고, 리포트, 검수 로그 |

## 승격 흐름

```text
철학
-> 영구 유지 체계 / 헌법 / 지도 후보
-> 기능 지도 또는 개발 지도
-> 발주서
-> 실행 기록
```

## 금지

1. 철학 문서를 곧바로 코드 실행 근거로 쓰지 않는다.
2. 사용자가 말을 끝내기 전에 코딩하지 않는다.
3. 발주서 없이 대규모 구현을 시작하지 않는다.
4. 실행 기록은 과거 기록이지 현재 명령이 아니다.

## 현재 개발 주기

앞으로 새 기능은 다음 주기로 진행한다.

```text
뚜렷한 목표 지정
-> MVP 구현
-> 학습 회고
```

세부 정책은 `01_Maintenance_System/DEVELOPMENT_CYCLE_POLICY_v0.md`에 둔다.
