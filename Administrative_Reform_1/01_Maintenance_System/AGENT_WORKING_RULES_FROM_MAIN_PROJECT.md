# Agent Working Rules From Main Project

## 목적

이 문서는 송련 본점 `SongRyeon_Project/AGENTS.md`의 핵심 지침을 SongRyeon Core 연습판의 영구 유지 체계에 이식한 문서다.

루트의 `AGENTS.md`는 Codex와 외부 협업자가 바로 읽는 작업 규칙이고,
이 문서는 송련의 내부 문서 검색 대상에 포함되도록 유지 체계 계층에 보존한다.

## 1. 한글 인코딩 규칙

SongRyeon Core의 `.md`, `.py`, `.json`, `.yml`, `.txt` 문서는 UTF-8을 기준으로 다룬다.

Windows PowerShell에서 한글이 깨져 보일 수 있다.
이 현상은 파일 자체가 깨졌다는 뜻이 아니라, 터미널 또는 읽기 명령의 인코딩 설정 문제일 수 있다.

문서를 읽을 때는 다음을 우선 사용한다.

```powershell
Get-Content path\to\file.md -Encoding UTF8
Get-Content path\to\file.md -Raw -Encoding UTF8
```

한글이 깨져 보이면 바로 문서를 다시 쓰지 않는다.
먼저 UTF-8로 다시 읽고, 실제 파일 손상인지 확인한다.

## 2. 사람과 AI의 역할 분담

정후는 비전, 구조 철학, 최종 결재를 담당한다.
Codex는 코드 작성, 수정, 테스트, 실행 기록을 담당한다.
송련 런타임 노드와 LLM은 주어진 입력과 근거 안에서 보고를 생성한다.

AI 협업자는 사용자의 비전 결정을 대신하지 않는다.
정책 판단이 필요한 경우에는 질문하거나 문서화한다.

## 3. 실행 증거와 설계 권한의 구분

런타임 데모 출력, 사용자 재현 로그, 테스트 캡처는 중요한 증거다.
하지만 증거가 곧바로 설계 권한은 아니다.

코드 변경이 필요하면 다음 중 하나로 승격한다.

- 명시적 사용자 요청
- 발주서
- 유지 체계 문서
- 기능/개발 지도
- 실행 기록에서 확인된 후속 작업

## 4. 메타정보 관리 원칙

송련은 절대 정보, 상대 정보, 혼합 정보를 구분한다.

절대 정보는 코드와 도구가 확정할 수 있는 ID, 경로, 존재 여부, 횟수, schema 통과 여부, tool result다.

상대 정보는 특정 하나의 절대 정보 record/field에 대응하거나 그것을 근거로 생성된 LLM의 해석, 의미 판단, 목표 달성 평가, 요약이다.

혼합 정보는 LLM 해석이 여러 절대 정보 묶음에 근거하거나, 특정 하나의 절대 정보로 지정하는 것이 불가능하거나 부적절할 때 쓰는 source-bundle 정보다.

코드는 의미 판단을 한 것처럼 위장하지 않는다.
LLM 판단은 `generated_by`, `info_class`, `semantic_judgement_status`, `source_data_ids`로 드러낸다.

## 5. 안전한 작업 규칙

- 사용자 변경을 되돌리지 않는다.
- 무관한 파일을 삭제하거나 이동하지 않는다.
- 대량 구조 변경, 라우팅 변경, 프롬프트 변경, DB 변형, answer mode 변경은 먼저 문서화하거나 결재를 받는다.
- 테스트 결과와 실패 원인을 실행 기록에 남긴다.
- 같은 실수를 반복하면 유지 체계 문서에 규칙을 추가한다.

## 현재 적용 이유

2026-06-24 테스트 중 PowerShell 출력에서 한글 모지바케가 반복적으로 관찰되었다.
또한 런타임 예산, LLM 판단, 코드 판단이 섞여 보일 위험이 있었으므로 본점 지침을 Core 연습판에도 고정한다.
