# Development Map v0

## 목적

이 문서는 SongRyeon Core를 구현할 때의 순서를 정리한다. 아직 발주서가 아니며, 코딩은 사용자가 명시적으로 시작하라고 할 때만 진행한다.

## 권장 개발 순서

1. 설계 문서를 먼저 확정한다.
2. 스키마만 만든다.
3. trace 기록기만 만든다.
4. LLM 없는 0 기억공급관 가짜 버전을 만든다.
5. LLM 없는 1 라우터 가짜 버전을 만든다.
6. LLM 없는 2 메타정보 경계관 가짜 버전을 만든다.
7. 3 보고관으로 Markdown 보고서를 만든다.
8. 그 다음에 Qwen 14B를 붙인다.
9. 마지막에 L루프와 내부 문서 검색을 붙인다.

## 학습 포인트

### 스키마

머릿속 개념을 코드의 그릇으로 바꾸는 단계다.

### Trace

일 돌아가는 흐름을 재현 가능한 데이터로 남기는 단계다.

### 0

기억을 창조하지 않고, trace와 state에서 필요한 기억을 잘라 공급하는 단계다.

### 1

상황 판단을 라우팅과 라우팅 이유 작성으로 제한하는 단계다.

### 2

절대 정보, 상대 정보, 혼합 정보의 경계를 강제하는 단계다.

### 3

2가 허용한 정보 안에서만 말하는 단계다.

## 아직 발주하지 않을 것

- Qwen 14B 연결.
- 그래프 DB.
- 장기기억 벡터 검색.
- C/R/W/M 루프.
- L3의 재진입/빠꾸 로직.
