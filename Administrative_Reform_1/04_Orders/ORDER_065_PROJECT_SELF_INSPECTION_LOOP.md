# ORDER 065: Project Self-Inspection Loop

**상태**: 정식 발주서  
**승격일**: 2026-06-22  
**출처**: 발주서, 실행 기록, 코드 구현 상태가 빠르게 늘어나는 상황  
**목표**: 프로젝트가 자기 문서를 검색해 현재 구현 상태와 다음 작업 후보를 보고하는 자기점검 루프를 만든다.

## 배경

발주서가 60개를 넘으면 사용자가 매번 전체 구조를 손으로 따라가기 어렵다.  
연습장은 자기 문서와 실행 기록을 읽고, 무엇이 구현되었고 무엇이 아직 발주서 상태인지 말할 수 있어야 한다.

## 범위

1. `SelfInspectionFrame` 또는 동등한 구조를 만든다.
2. 점검 대상은 발주서, 실행 기록, 지도 문서, 주요 코드 파일이다.
3. 점검 결과에는 `implemented`, `planned`, `stale`, `blocked`, `recommended_next`를 둔다.
4. 추천 작업은 반드시 근거 문서 ID 또는 파일 경로를 가져야 한다.
5. 자기점검 루프는 코드를 자동 수정하지 않고 보고서만 만든다.

## 원칙

1. 자기점검은 추천이지 자율 변경 권한이 아니다.
2. 구현 여부 판단은 문서만 보지 말고 코드와 실행 기록을 함께 봐야 한다.
3. 오래된 문서는 폐기하지 말고 stale 후보로 표시한다.

## 완료 기준

1. 자기점검 probe가 다음 작업 후보를 보고한다.
2. 보고서가 발주서와 실행 기록을 근거로 인용한다.
3. 구현 완료/미완료/문서 노후화 후보가 구분된다.
4. `python main.py smoke-test`가 통과한다.
