# ORDER 005-012: Dry Run Stack

**상태**: 정식 발주서  
**승격일**: 2026-06-21  
**출처**: `TMP_Auto_Hunt_2026_06_21/TMP_ORDER_005`부터 `TMP_ORDER_012`까지 일괄 승격  
**목표**: LLM 없이 `0 -> 1 -> 0 -> L -> 0 -> 1 -> 0 -> 2 -> 3` 흐름이 데이터로 도는 드라이런 스택을 만든다.

## 범위

1. 0 기억공급관 규칙 기반 버전.
2. 1 라우터 규칙 기반 버전.
3. 2 메타정보 경계관 절대정보 버전.
4. 3 보고관 Markdown 버전.
5. L1/L2/L3 규칙 기반 루프.
6. 프롬프트/스키마 레지스트리 최소 구조.
7. 실패 신호 helper.
8. 드라이런 실행 파일.

## 원칙

1. Qwen 14B는 연결하지 않는다.
2. LangGraph는 연결하지 않는다.
3. DB나 벡터 검색은 붙이지 않는다.
4. 상대 정보와 혼합 정보는 최소화하고, trace/data id 중심으로 연결한다.
5. 드라이런은 구조 검증용이다.

## 완료 기준

1. `python -m py_compile`이 통과한다.
2. `python dry_run.py`가 실행된다.
3. trace, state, zero_state capsule, node movement, 3번 보고서가 출력된다.
