# ORDER 059: Memory Insufficient Backpressure

**상태**: 정식 발주서  
**승격일**: 2026-06-22  
**출처**: 사용자가 지적한 "0이 접근 못하는 상황" 문제  
**목표**: 0이 정보 부족을 감지하면 다음 노드로 부실한 기억을 넘기지 않고 반려/재라우팅 신호를 낸다.

## 배경

초기 설계에서 사용자는 0이 접근 못하는 상황이 생기면 구조적 문제가 생긴다고 지적했다.  
MVP에서는 L루프가 장기기억을 검색하지만, 나중에는 C루프나 다른 복구 루프로도 연결될 수 있다.

## 범위

1. `MemoryInsufficientFrame` 또는 FailureSignal 확장을 만든다.
2. 0은 필요한 source가 없거나 read window가 부족하면 부족 신호를 낸다.
3. 1은 부족 신호를 보고 L/C/사용자 질문/중단 중 하나로 라우팅할 수 있다.
4. 부족 신호에는 필요한 정보 종류와 현재 가진 근거 ID를 적는다.
5. 같은 부족이 반복되면 사용자에게 명확히 보고한다.

## 원칙

1. 부족한 기억을 그럴듯하게 채우지 않는다.
2. 부족 신호는 실패가 아니라 복구 가능한 상태다.
3. 0의 반려는 1의 라우팅 판단 재료가 된다.

## 완료 기준

1. memory packet에 insufficient signal이 실제로 연결된다.
2. 1이 insufficient signal을 읽을 수 있다.
3. 부족 상황 smoke test가 실패 대신 복구 경로를 기록한다.
4. 기존 정상 dry run은 깨지지 않는다.
