# TMP ORDER 006: Node 1 Router

**상태**: 임시 발주서  
**목표**: 1 상황판단 라우터의 입력, 출력, 라우팅 기준을 정의한다.  
**실행 권한**: 없음.

## 배경

1은 최근 대화, 사용자 직전 입력, 0의 보고를 듣고 상황을 판단한다. 판단력은 라우팅과 라우팅 이유 작성에 몰빵한다.

## MVP 라우팅 대상

```text
2:
메타정보 경계관. 사용자에게 보고하기 전 사실/상대/혼합 정보의 선을 그을 때 선택한다.

L:
장기기억과 내부문서를 검색해야 할 때 선택한다.
```

## 출력

```text
RoutingDecision
- route
- routing_reason
- required_schema
- target_profile
- expected_next_0_mode
- uncertainty
```

## 필드 설명

```text
required_schema:
다음 노드에 강제할 스키마 이름과 강제 여부.

expected_next_0_mode:
1의 라우팅 이후 0이 어떤 모드로 다시 호출되어야 하는지.
```

## 완료 기준

- 1이 직접 일을 처리하지 않고 라우팅에 집중한다는 원칙이 적혀 있다.
- 라우팅 이유가 0의 조건부 프롬프트 재료가 된다는 점이 적혀 있다.
- 2와 L 중 어디로 보낼지 최소 기준이 있다.

## 제외

- 복잡한 라우팅 정책.
- R/W/C/M 루프.
