# TMP ORDER 012: Dry Run MVP

**상태**: 임시 발주서  
**목표**: LLM 없이도 한 턴이 데이터로 흐르는 드라이런 MVP를 만든다.  
**실행 권한**: 없음.

## 배경

처음부터 Qwen 14B를 붙이면 모델 문제인지 구조 문제인지 구분하기 어렵다. 먼저 LLM 없이 흐름이 도는지 확인해야 한다.

## 목표 흐름

```text
사용자 입력
-> trace 기록
-> 0 가짜 기억공급
-> 1 가짜 라우팅
-> 0 조건부 기억공급
-> 2 가짜 메타정보 경계
-> 3 Markdown 보고
-> 턴 상태 캡슐 생성
```

## 만들 것

- 샘플 사용자 입력 1개.
- trace event 목록.
- 0 memory packet 샘플.
- 1 routing decision 샘플.
- 2 metainfo boundary 샘플.
- 3 report 샘플.
- turn state capsule 샘플.

## 학습 목표

이 작업은 코딩을 배우기 위한 첫 실행 후보가 될 수 있다. 핵심은 LLM의 똑똑함이 아니라 데이터 흐름을 눈으로 확인하는 것이다.

## 완료 기준

- 터미널 또는 문서에서 한 턴의 흐름이 보인다.
- 각 단계의 입력과 출력이 trace로 남는다.
- Qwen 14B 없이도 구조가 이해된다.

## 제외

- 실제 LLM 호출.
- LangGraph.
- 내부 문서 검색.
