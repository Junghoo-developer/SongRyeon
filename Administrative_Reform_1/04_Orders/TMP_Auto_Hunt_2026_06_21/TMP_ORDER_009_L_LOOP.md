# TMP ORDER 009: L Loop

**상태**: 임시 발주서  
**목표**: 장기기억/내부문서 검색 루프인 L루프의 L1, L2, L3 역할을 정의한다.  
**실행 권한**: 없음.

## 배경

L루프는 0의 기억만으로 부족하거나 내부 문서에서 근거를 찾아야 할 때 실행된다.

## 흐름

```text
1이 L로 라우팅
-> 0이 L루프에 필요한 기억 공급
-> L1
-> L2
-> L3
-> 0이 L루프 trace를 1에게 요약
-> 1 복귀
```

## L1

0에게 보고를 받고 처음 시동을 건다.

출력:

- 거시 목표.
- 미시 목표.
- 목표 설정 이유.

## L2

0의 보고와 L1의 출력을 보고 검색어를 설정한다.

출력:

- 검색어 후보.
- 각 검색어의 이유.
- 검색 대상 힌트.

## L3

L1, L2의 출력과 그 이유에 대응하여 달성 여부를 판단한다. 동시에 전달해야 할 정보를 적절한 양으로 보존한다.

출력:

- 거시 목표 달성 여부.
- 미시 목표 달성 여부.
- 달성 판단 이유.
- 보존할 정보.
- 출처 절대 정보.

## 나중 확장

L3이 빠꾸를 먹이거나, 목표 달성 여부에 따라 L1로 되돌아가는 구조는 나중에 다룬다. 현재 MVP에서는 제외한다.

## 완료 기준

- L1, L2, L3의 역할이 구분되어 있다.
- 0이 L 전후에 끼어드는 위치가 정의되어 있다.
- L3의 달성 판단과 보존 정보가 정의되어 있다.

## 제외

- 실제 검색 엔진.
- 벡터 DB.
- L3 재진입 분기.
